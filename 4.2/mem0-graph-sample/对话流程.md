# mem0 图谱记忆对话系统 - 流程梳理

## 系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Neo4j     │     │   Qdrant    │     │   Redis     │
│  (图数据库)  │     │ (向量数据库) │     │   (缓存)    │
└─────────────┘     └─────────────┘     └─────────────┘
       ↑                   ↑                    ↑
       └───────────────────┴────────────────────┘
                           │
                    ┌──────▼──────┐
                    │  mem0 Memory│
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  对话系统    │
                    └─────────────┘
```

## 记忆召回流程（何时从缓存读，何时从数据库读）

### 场景一：带缓存的系统（mem0_chat_with_cache.py）

```
用户发送消息
   │
   ▼
需要检索记忆
   │
   ▼
┌─────────────────────────────────────┐
│ 生成缓存 Key                         │
│ Key = "mem0:search:{user_id}:{query_hash}" │
└─────────────────────────────────────┘
   │
   ▼
查询 Redis 缓存
   │
   ├─────────────────┬─────────────────┐
   │                 │                 │
   ▼                 ▼                 ▼
缓存命中           缓存未命中         缓存损坏
   │                 │                 │
   │ 直接返回结果      │ 查询数据库       │ 删除缓存，查询数据库
   │ (快速响应)       │                 │
   │                 ▼                 ▼
   │         ┌─────────────────────┐
   │         │ 查询 Qdrant 向量库   │
   │         │ 进行向量相似度搜索   │
   │         └─────────────────────┘
   │                 │
   │                 ▼
   │         获取搜索结果
   │                 │
   │                 ▼
   │         ┌─────────────────────┐
   │         │ 回写 Redis 缓存      │
   │         │ 设置 TTL (1小时)     │
   │         └─────────────────────┘
   │                 │
   └─────────────────┴─────────────────┘
                     │
                     ▼
返回记忆结果
```

**关键点：**
- **缓存命中**：直接从 Redis 读取，无需查询数据库
- **缓存未命中**：查询 Qdrant 数据库，并将结果写入缓存
- **缓存策略**：Cache-Aside 模式（旁路缓存）

### 场景二：图谱记忆系统（mem0_graph_chat_simple.py，无缓存）

```
用户发送消息
   │
   ▼
需要检索记忆
   │
   ▼
┌─────────────────────────────────────┐
│ 并行查询数据库                        │
└─────────────────────────────────────┘
   │
   ├─────────────────┬─────────────────┐
   │                 │                 │
   ▼                 ▼                 ▼
查询 Qdrant         查询 Neo4j        合并结果
向量数据库          图数据库
   │                 │                 │
   │ 向量相似度搜索    │ 查询实体关系      │
   │ 返回相似记忆      │ 返回关系图谱      │
   │                 │                 │
   └─────────────────┴─────────────────┘
                     │
                     ▼
返回记忆结果（记忆 + 关系）
```

**关键点：**
- **无缓存层**：每次查询都直接访问数据库
- **双重查询**：同时查询向量库和图库
- **实时数据**：始终获取最新数据，但查询较慢

## 记忆添加流程（何时存数据库，何时失效缓存）

### 场景一：带缓存的系统

```
用户退出对话
   │
   ▼
检查是否有对话记录
   │
   ├─→ [无记录] → 直接退出
   │
   └─→ [有记录] ↓
   
   │
   ▼
┌─────────────────────────────────────┐
│ 保存记忆到数据库                      │
│ memory.add(conversation_history)     │
└─────────────────────────────────────┘
   │
   ├─→ 提取实体和关系
   │
   ├─────────────────┬─────────────────┐
   │                 │                 │
   ▼                 ▼                 ▼
写入 Qdrant         写入 Neo4j        保存完成
向量数据库          图数据库
   │                 │                 │
   │ 存储记忆向量      │ 存储实体关系      │
   │                 │                 │
   └─────────────────┴─────────────────┘
                     │
                     ▼
┌─────────────────────────────────────┐
│ 失效相关缓存                          │
│ 清除该用户的所有查询缓存               │
└─────────────────────────────────────┘
   │
   ├─→ 查找缓存 Key 模式: "mem0:search:{user_id}:*"
   │
   └─→ 删除所有匹配的缓存 Key
   
   │
   ▼
保存完成
```

**关键点：**
- **先存数据库**：确保数据持久化
- **后失效缓存**：确保后续查询获取最新数据
- **批量失效**：清除该用户的所有查询缓存（因为新记忆可能影响所有查询）

### 场景二：图谱记忆系统（无缓存）

```
用户退出对话
   │
   ▼
检查是否有对话记录
   │
   ├─→ [无记录] → 直接退出
   │
   └─→ [有记录] ↓
   
   │
   ▼
┌─────────────────────────────────────┐
│ 保存记忆到数据库                      │
│ memory.add(conversation_history)     │
└─────────────────────────────────────┘
   │
   ├─→ 自动提取实体
   │
   ├─→ 自动提取关系
   │
   ├─────────────────┬─────────────────┐
   │                 │                 │
   ▼                 ▼                 ▼
写入 Qdrant         写入 Neo4j        保存完成
向量数据库          图数据库
   │                 │                 │
   │ 存储记忆向量      │ 存储实体关系      │
   │                 │                 │
   └─────────────────┴─────────────────┘
                     │
                     ▼
保存完成（无需处理缓存）
```

**关键点：**
- **直接保存**：无需考虑缓存失效
- **自动提取**：mem0 自动提取实体和关系
- **双重存储**：同时写入向量库和图库

## 完整对话流程（带缓存）

```
程序启动
   │
   ▼
初始化系统
   ├─→ 连接 Qdrant 向量数据库
   ├─→ 连接 Redis 缓存
   └─→ 配置大语言模型
   
   │
   ▼
┌─────────────────────────────────────┐
│ 进入对话循环                         │
└─────────────────────────────────────┘
   │
   ▼
用户发送消息
   │
   ▼
┌─────────────────────────────────────┐
│ 召回记忆（带缓存）                    │
└─────────────────────────────────────┘
   │
   ├─→ 查询 Redis 缓存
   │     ├─→ [命中] → 直接返回
   │     └─→ [未命中] → 查询 Qdrant → 回写缓存
   │
   ▼
构建上下文
   │
   ▼
调用大语言模型生成回答
   │
   ▼
显示回复
   │
   └─→ 继续对话循环
   
   │
   ▼
用户退出
   │
   ▼
┌─────────────────────────────────────┐
│ 添加记忆                              │
└─────────────────────────────────────┘
   │
   ├─→ 保存到 Qdrant 数据库
   │
   └─→ 失效 Redis 缓存（清除该用户所有查询缓存）
   
   │
   ▼
程序结束
```

## 缓存策略对比

| 场景 | 召回记忆 | 添加记忆 | 优点 | 缺点 |
|------|---------|---------|------|------|
| **带缓存系统** | 先查 Redis，未命中查 Qdrant | 存 Qdrant + 失效 Redis 缓存 | 查询速度快，减少数据库压力 | 需要维护缓存一致性 |
| **图谱记忆系统** | 直接查 Qdrant + Neo4j | 存 Qdrant + Neo4j | 数据实时，无需维护缓存 | 查询较慢，数据库压力大 |

## 关键时机总结

### 何时从缓存读？
- ✅ **带缓存系统**：每次用户发送消息时，先查询 Redis 缓存
- ✅ **缓存命中**：直接返回缓存结果，无需查询数据库
- ❌ **图谱记忆系统**：无缓存，每次都查数据库

### 何时从数据库读？
- ✅ **带缓存系统**：缓存未命中时，查询 Qdrant 向量数据库
- ✅ **图谱记忆系统**：每次用户发送消息时，查询 Qdrant 和 Neo4j

### 何时存数据库？
- ✅ **用户退出时**：将完整对话历史保存到数据库
- ✅ **带缓存系统**：保存到 Qdrant
- ✅ **图谱记忆系统**：保存到 Qdrant 和 Neo4j

### 何时失效缓存？
- ✅ **带缓存系统**：保存新记忆后，立即清除该用户的所有查询缓存
- ✅ **失效原因**：新记忆可能影响所有查询结果，必须保证数据一致性
- ❌ **图谱记忆系统**：无缓存，无需失效
